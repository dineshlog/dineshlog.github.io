Identity & Access Management
Microsoft Entra ID configuration baselines, Conditional Access policies, PIM, MFA enforcement, and external identity controls aligned to NIST CSF, CIS Controls v8.1, CIS M365 Benchmark §1, ISO 27001, and GDPR.
🚨
Critical: Identity is the #1 attack vector in M365. Password spray and phishing account for 99%+ of credential compromises. MFA and phishing-resistant auth (FIDO2/Passkeys/WHfB) are P0 controls. Block legacy auth before all else.

Implementation Roadmap & Audit Evidence
Phased implementation approach, operating cadence, audit evidence pack, and source references.
Prioritised Recommendations — Critical (Implement Immediately)
CriticalR-01 · Enforce MFA for 100% of users with phishing-resistant methods
Deploy CA policies requiring MFA for all users. Migrate admins to FIDO2/WHfB. Disable legacy per-user MFA. Target: 0% admin exemptions.
NIST PR.AA-02ISO A.8.5CIS 6.3CIS M365 §1.1.1GDPR Art.32
CriticalR-02 · Block legacy authentication protocols globally
CA policy to block all legacy auth. Run sign-in logs filtered by legacy client first. 2-week report-only, then enforce. SMTP AUTH disabled tenant-wide.
NIST PR.AA-05ISO A.8.20CIS 4.2CIS M365 §1.1, §2.5
CriticalR-03 · Deploy DMARC at p=reject for all sending domains
Audit all sending domains. Configure SPF (hard fail), DKIM, DMARC progression (none → quarantine → reject over 60 days). Register with DMARC reporting service.
NIST PR.DS-02ISO A.8.23CIS 9.1CIS M365 §2.2.1–2.2.3GDPR Art.32
CriticalR-04 · Enable Purview Audit Premium + export to Sentinel
Upgrade to Audit Premium. Retain ≥1 year. Enable MailItemsAccessed, Send, SearchQueryInitiatedExchange. Export to Sentinel for SIEM. Maps to GDPR Art.30 records obligation.
NIST DE.AE-02ISO A.8.15CIS 8.2CIS M365 §5.1GDPR Art.30
CriticalR-05 · Run DSPM for AI + remediate oversharing before Copilot
Before any Copilot expansion: run DSPM for AI reports, remediate "Anyone" sharing links, label all confidential content, restrict overshared SharePoint sites. Copilot respects existing permissions — over-permissioned data becomes AI-accessible.
NIST PR.DSISO A.8.12CIS M365 §3.1GDPR Art.5EU AI Act Art.10
